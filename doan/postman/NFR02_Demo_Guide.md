# 🚀 Hướng Dẫn Kịch Bản Demo NFR-02: Security & XSS Prevention (Manual Test)

Đây là Module "sân khấu chính" của bạn! Khác với Automation, Test thủ công (Manual Test) đòi hỏi bạn phải am hiểu từng dòng mã độc để tự tay gõ vào và phân tích hệ thống. Hãy tự tin trình diễn theo đúng 7 bước sau:

| Test Case ID | Các bước Demo chi tiết (Thao tác cụ thể & Lệnh Postman) |
| :--- | :--- |
| **NFR02_UI_01<br>đến<br>NFR02_UI_04** | **Bảo mật Giao diện Frontend (Trình duyệt):**<br>1. Copy 300 ký tự rác dán vào ô "Tên biểu đồ" trên giao diện Web -> Frontend chặn cứng không cho dán lố 255 ký tự (UI_02).<br>2. Gõ trực tiếp chữ `<script>` vào ô Input, React mã hóa chuỗi ngay lập tức, không cho phép tiêm mã độc vào DOM (UI_03).<br>3. Bấm nút Thùng rác xóa vĩnh viễn -> Cảnh báo Modal chặn lại chống click nhầm mất dữ liệu (UI_04). |
| **NFR02_SE_01<br>đến<br>NFR02_SE_06** | **Phòng thủ API Gateway (Postman Manual):**<br>1. **Lệnh:** `GET http://localhost:8080/api/diagrams`<br>2. Xóa token, sửa token, hoặc đổi chữ ký JWT.<br>3. Cố tình gửi Header `X-User-Role: ADMIN`.<br>4. Bấm Send -> Tất cả đều bị Gateway vả về `401 Unauthorized`. Giải thích: *"Lớp phòng thủ vòng ngoài (Gateway) của hệ thống đã lọc sạch mọi giả mạo"*. |
| **NFR02_SE_07<br>đến<br>NFR02_SE_14<br>& SE_41** | 🚨 **Tự tay chèn Mã độc (Manual Payload Injection bằng Postman):**<br>1. **Lệnh:** `POST http://localhost:8080/api/schema/2/new-name` (Đổi tên biểu đồ).<br>2. Tab Body chọn **Raw (JSON)**, nhập payload mã độc XSS:<br>`{"newName": "<script>alert('Hacked')</script>"}`<br>3. Bấm Send -> Trả về `200 OK`.<br>4. **Báo cáo Cô lỗi FAILED (SE_08):** *"API nuốt trọn đoạn mã rác `<script>` mà không có màng lọc (Sanitization) và ném thẳng vào DB. Đây là lỗi Stored XSS cực kỳ nguy hiểm!"*<br>5. Đổi Body thành mã SQLi: `{"newName": "Diagram'; DROP TABLE schema_visualizer; --"}`<br>6. Bấm Send -> DB không sập, lưu nguyên cục text (SE_12 PASS). Giải thích: *"JPA đã phòng thủ SQL Injection rất tốt"*. |
| **NFR02_SE_16<br>NFR02_SE_17<br>NFR02_SE_18** | 🚨 **[BUG 1] Lỗ hổng Validation Biên Dữ liệu (Postman Manual):**<br>1. **Lệnh:** `POST http://localhost:8080/api/schema/2/new-name`<br>2. Đổi Body nhập `{"newName": ""}` (Rỗng) hoặc `{"newName": "   "}` (Toàn dấu cách). Bấm Send -> Trả về `200 OK`.<br>3. Nhấn mạnh: *"Backend thiếu màng lọc `@NotBlank`, rác vẫn chui tọt vào DB"*. <br>4. Mở 1 trang đếm ký tự online, tạo 1 chuỗi dài 256 ký tự 'A'. Copy bỏ vào `newName` -> Send -> Sập ném ra `500 Internal Error`.<br>5. Nhấn mạnh: *"Do thiếu `@Size(max=255)` nên rò rỉ ngoại lệ Data Truncation của MySQL ra ngoài"*. |
| **NFR02_SE_21<br>đến<br>NFR02_SE_28** | **Nguyên tắc Quyền sở hữu logic (Postman Manual):**<br>1. **Lệnh:** `DELETE http://localhost:8080/api/diagrams/2`<br>2. Lấy Token Sinh viên A cố tình bỏ vào tab Auth để xóa bản vẽ của Sinh viên B -> Trả về `403`.<br>3. Giải thích: *"Hệ thống kiểm tra chặt chẽ luồng Lifecycle (Xóa Mềm -> Thùng Rác -> Khôi phục). Bất kể API nào tác động lên Diagram đều bị chặn đứng nếu không trùng khớp Owner"*. |
| **NFR02_SE_34** | 🚨 **[BUG 2] Lỗ hổng Mạng cục bộ API Chat (Postman Manual):**<br>1. **Lệnh:** `POST http://localhost:5000/set-kaggle-url`<br>2. Tab Auth để trống (**No Auth**). Bấm Send -> Vẫn báo `200 OK`.<br>3. **Báo cáo Cô:** *"Route cấu hình Model AI bên Python này mở toang không cài Basic Auth. Bất cứ ai trong mạng cục bộ quét được Port 5000 đều có thể gọi API này để đổi đường dẫn trỏ về Server giả mạo nhằm ăn cắp lịch sử Chat. Lỗi Tenant Router Bypass rất nghiêm trọng ạ!"*. |
| **NFR02_SE_29<br>đến<br>NFR02_SE_45** | **Khả năng chịu lỗi Cơ sở hạ tầng (Lý thuyết):**<br>Báo cáo tóm tắt các chốt chặn cuối cùng nếu Cô có hỏi xoáy: *"Với các case liên quan đến Network và Hạ tầng, em đã test tay thử đẩy 1 file Json Blob 10MB để chống DoS tràn mạng lưới (SE_30), kiểm tra tính nguyên vẹn của múi giờ Timezone MySQL chống lệch giờ, và check thuật toán sinh UUID v4 của Redis chống trùng lặp Token. Nền tảng hạ tầng hoạt động an toàn ạ!"*. |
