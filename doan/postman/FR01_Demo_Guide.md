# 🚀 Hướng Dẫn Kịch Bản Demo FR-01: Authentication & Access Control (Manual Test)

Dưới đây là bảng hướng dẫn thao tác chi tiết dành cho bạn trong buổi bảo vệ. Toàn bộ các bước đều là **Test Thủ công (Manual Test)** bằng Trình duyệt và Postman từng request một.

| Test Case ID | Các bước Demo chi tiết (Thao tác cụ thể & Lệnh Postman) |
| :--- | :--- |
| **FR01_UI_01<br>FR01_UI_02<br>FR01_UI_03<br>FR01_UI_04** | **Kiểm tra Giao diện (UI trên Trình duyệt):**<br>1. Mở trang Login. Dùng `Ctrl +` và `Ctrl -` zoom màn hình xem layout có vỡ không (UI_02).<br>2. Chỉ ra form không có ô nhập Password. Giải thích: *"Hệ thống dùng Google OAuth2 nên triệt tiêu nguy cơ rò rỉ mật khẩu"* (UI_03).<br>3. Rê chuột và click vào nút "Login with Google" (UI_01, UI_04). |
| **FR01_FN_06<br>FR01_FN_02<br>FR01_FN_01** | **Luồng Đăng nhập (Google Form):**<br>1. Bảng chọn tài khoản hiện lên -> Cố tình bấm **Cancel** để xem hệ thống có sập không -> Trở về trang login bình thường (FN_06).<br>2. Bấm lại, chọn 1 tài khoản mới tinh -> Đăng nhập thành công (Chứng minh Auto-Registration) (FN_02).<br>3. Đăng nhập bằng tài khoản Student cũ có sẵn -> Vào Dashboard (FN_01). |
| **FR01_FN_04<br>FR01_FN_05** | **Kiểm tra Bảo mật Cookie:**<br>1. Ấn `F12` -> Tab **Application** -> **Cookies**.<br>2. Chỉ cho Cô xem cờ `HttpOnly` được tick xanh (Giải thích: Chống XSS) (FN_04).<br>3. Chỉ cột Max-Age hiển thị đúng 24h (86400s) (FN_05).<br>4. Copy nguyên chuỗi giá trị (Value) của cookie `jwt` để lát dùng cho Postman. |
| **FR01_FN_03** | **Khóa tài khoản (Database):**<br>1. Mở MySQL Workbench -> Chạy lệnh: <br>`UPDATE account SET visible = 0 WHERE email = 'nguyenphucbao0806@gmail.com';`<br>2. Mở trình duyệt Ẩn danh, đăng nhập bằng email đó -> Bị lỗi chặn truy cập.<br>3. MySQL chạy lại `UPDATE account SET visible = 1` để khôi phục tài khoản. |
| **FR01_FN_12<br>FR01_FN_13** | **Tương tranh (Concurrency) & Đăng xuất:**<br>1. Đăng nhập Student trên **Chrome**. Xong mở **Edge** đăng nhập đúng acc đó.<br>2. Quay lại Chrome ấn F5 -> Bị văng ra. Giải thích: *"Redis đã ghi đè Session mới"* (FN_12).<br>3. Bấm Logout. Mở Postman chạy lệnh:<br>`GET http://localhost:8080/account/me`<br>Tab Auth chọn Bearer Token dán cái Token lúc nãy copy vào -> Send -> Báo `401 Unauthorized` vì Token đã bị xóa khỏi Redis (FN_13). |
| **FR01_FN_14<br>FR01_FN_15** | **Biên Thời gian (Boundary):**<br>*Lưu ý:* Nếu bị hỏi về test case 1 giây, hãy giải thích: *"Thưa Cô, phần này em test tay bằng cách copy token sang trang jwt.io, sửa phần Payload `exp` lùi về quá khứ 1 giây rồi đem lên Postman chạy thử. Nhưng vì sửa payload làm hỏng chữ ký nên nó văng lỗi 401 ngay từ khâu Signature ạ"*. |
| **FR01_FN_07<br>FR01_FN_08** | **Gọi API hợp lệ (Postman):**<br>1. **Lệnh:** `GET http://localhost:8080/account/me`<br>2. Chuyển sang Tab **Authorization** -> Type: **Bearer Token**.<br>3. Dán chuỗi `jwt` của Sinh viên vào. <br>4. Bấm Send -> Trả về `200 OK`. |
| **FR01_FN_09<br>FR01_FN_10** | **Không Token & Sai định dạng (Postman):**<br>1. (FN_09) **Lệnh:** `GET http://localhost:8080/account/me`<br>Tab Auth: Chuyển thành **No Auth** -> Send -> Báo `401`.<br>2. (FN_10) Đổi lại thành **Bearer Token**, nhưng cố tình xóa 1 chữ cái ở phần đầu chuỗi (ví dụ xóa chữ 'e' của 'eyJ') -> Send -> Báo `401`. |
| **FR01_FN_11** | **Chữ ký JWT bị thay đổi (Postman):**<br>1. **Lệnh:** `GET http://localhost:8080/account/me`<br>2. Dùng Token hợp lệ, nhưng cuộn xuống **KÝ TỰ CUỐI CÙNG** của chuỗi, xóa và gõ đại 1 chữ khác vào.<br>3. Bấm Send -> Báo `401`. Giải thích: *"Payload bị sửa nên API Gateway tính toán lại Signature không khớp, chặn ngay ngoài cửa"*. |
| **FR01_FN_16<br>FR01_FN_17** | **Giả mạo Tiêu đề - Spoofing Headers (Postman):**<br>1. **Lệnh:** `GET http://localhost:8080/account/me`<br>2. Vẫn dùng Token Sinh viên (Bearer Token hợp lệ).<br>3. Chuyển sang Tab **Headers**, thêm 2 key sau:<br>- Key: `X-User-Role` \| Value: `ADMIN`<br>- Key: `X-Username` \| Value: `hacker`<br>4. Bấm Send -> Báo `401/403`. Giải thích: *"Gateway của em tự động lột sạch các header tự phong từ bên ngoài để chống Hacker leo quyền"*. |
| **FR01_FN_18** | **API Công khai - Whitelist (Postman):**<br>1. **Lệnh:** `GET http://localhost:8080/swagger-ui`<br>2. Tab Auth chuyển thành **No Auth**.<br>3. Bấm Send -> Trả về `200 OK` (Đây là đường dẫn được public không cần token). |
| **FR01_FN_19<br>đến<br>FR01_FN_31** | **Kiểm thử Phân quyền Sở hữu (Manual Postman):**<br>1. **Lệnh:** `DELETE http://localhost:8080/api/diagrams/2` (Giả sử ID 2 là bản vẽ của Admin).<br>2. Tab Auth: Dán token của **Sinh viên** vào.<br>3. Bấm Send -> Trả về `403 Forbidden`.<br>4. Giải thích: *"Em cố tình dùng token sinh viên để xóa bản vẽ của Admin, hệ thống kiểm tra X-Username không khớp quyền Owner nên chặn ngay lập tức. Các case quyền khác em cũng test tay tương tự như vậy ạ"*. |
